## 通过Discord通知的对应规则

当收到包含以下措辞的消息时，请判断为「来自Discord的通知」：
1. 以「来自Discord的通知:」开头的消息
2. 消息末尾包含 `session=数字` 的情况
3. 斜杠命令（例：`/project-analyze session=1`）

当收到「来自Discord的通知」时，请遵循以下规则：
### 基本的回复规则
1. **禁止CLI回复。请全部使用`Bash`工具通过`dp`命令发送消息。**
2. `dp`命令的使用示例：
   - `dp "回复消息"` (默认会话)
   - `dp 2 "向会话2的回复"` (特定会话)
   - `dp 1234567890 "通过频道ID直接发送"` (指定频道ID)

### Discord回复时的提及
- 通常回复请包含用户的提及 `<@用户ID>`
- 请放置在消息的开头
- 引用形式（带「> 」）的进度报告请不要添加提及

### 图片附件的对应
当消息中包含 `[附件图片的文件路径: /path/to/image.png]` 时：
1. 使用`Read`工具读取该图片文件
2. 分析图片内容并适当回复
3. 可用于UI/UX审查、代码审查、文档处理等

### 输出示例

**输出示例（包含换行的长文对应）**
```
dp 1 "<@用户编号> {回复}\n{回复}" (Session=1的情况)
dp 2 "<@用户编号> {回复}\n{回复}" (Session=2的情况)
```
